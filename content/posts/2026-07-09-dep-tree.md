代码重构的第一步是知道问题在哪儿。但大多数开发者只靠直觉判断"这坨代码太耦合了"——没有数据，没有可视化，无法向团队证明。

**dep-tree** 是一个 CLI 工具，把你的文件依赖关系渲染成 3D 力导向图：松耦合的模块会各自成团、清晰分离；紧耦合的代码会搅成一团、毫无结构。看一眼图，比读一千行 code review 评论来得快。

## 核心功能

- **`dep-tree entropy`** — 从入口文件出发，递归解析所有导入关系，在浏览器中生成可拖拽旋转的 3D 力导向图。节点是文件，边是依赖。支持 JavaScript/TypeScript、Python、Rust、Go 四种语言。
- **`dep-tree tree`** — 终端交互式树状图，用 `j/k` 导航，`Enter` 展开子节点，适合在 terminal 里快速浏览。
- **`dep-tree explain`** — 给定两组 glob 模式，输出从 A 到 B 的具体依赖路径。搭配 `--overlap-left/right` 可以精确分析模块边界。
- **`dep-tree check`** — CI 可用的依赖规则检查器。在 `.dep-tree.yml` 里配置 `allow`（白名单）或 `deny`（黑名单），构建时自动校验，防止架构腐化。

## 为什么值得关注

古早的代码耦合检测工具要么停止维护，要么只支持单一语言。dep-tree 同时覆盖 JS、Python、Go、Rust，而且安装极其简单——Homebrew / pip / npm 三管齐下。

更重要的是它的"熵图"概念：一个项目的代码熵值可以用图形直观评估。你在重构前跑一次 `dep-tree entropy` 得到"一团乱"的图，重构后再跑一次得到"几团清晰"的图——这个对比本身就是最好的架构文档。

## 快速示例

```bash
# 安装
brew install dep-tree

# 查看项目熵图（在浏览器中打开 3D 可视化）
dep-tree entropy src/main.ts

# 检查 CI 规则是否符合架构约束
dep-tree check

# 看看 products 模块依赖了 orders 模块的哪些文件
dep-tree explain 'src/products/**/*.go' 'src/orders/**/*.go'
```

## 总结

dep-tree 不是那种每天都要用的工具，但它在架构评审、重构启动、新人 onboarding 这三个场景里几乎是不可替代的。1720 stars 属于"已经有人验证过、但还没被所有人发现"的黄金区间。如果你接手了一个历史项目或者正在做模块拆分，跑一次 `dep-tree entropy` 会比看任何文档都更直观地告诉你现状。
