## 为什么需要 API 中转

同时用 OpenAI、Claude、DeepSeek 等多个模型，每个都要独立管理 API Key 和计费。sub2api 统一接管，一个地址走所有模型。

## 部署要点

### 服务端

sub2api 本质上是一个反向代理层，接收统一格式的请求，转发到不同后端：

- 部署在 HK 服务器上（国内直连快）
- 支持多模型路由，按需分配
- 自带用量统计和限速

### 客户端

一个关键配置：**不要在客户端写死模型名**。通过服务端配置文件统一管理，换模型只需改一处。

### 注意

- 模型名要和服务端配置严格一致，否则 404
- 虽然 sub2api 支持改名（比如 DeepSeek 伪装成 GPT），但**不建议这么做**——后面排查问题会很痛苦

## 用途

目前接入了 Telegram Bot、Hermes Agent 等客户端，统一走 sub2api 管理 API 调用和计费。
