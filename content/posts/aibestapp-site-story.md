## 项目定位

aibestapp.top 这个域名注册的时候，想法是做一个 AI 工具导航站。后来做的时候发现，纯粹的工具导航站太多了，再做也没有差异化。

于是调整了方向——把它做成一个"工具箱"，里面放一些自己开发的小工具和小功能。价值不一定大，但至少是原创的，跟那些搬运类网站有区别。

## 技术选择

你打开 aibestapp.top，会看到一个 3D 粒子动画的首页。那个动画是用 Three.js 写的，纯前端渲染，不需要后端支持。

选择 Three.js 不是因为网站需要 3D 效果，而是因为我想试试这个技术。做技术选型的时候，我倾向于在个人项目里用一些自己感兴趣的技术，即使从产品角度看不是最优选择。项目是你自己的，你有这个自由度。

其他页面都是纯 HTML/CSS/JS，没有任何框架依赖。好处很明显：编译不需要依赖打包工具，部署只需要把文件放到服务器目录下，浏览器的 F5 刷新就够用了。

## 目前的模块

| 页面 | 功能 |
|------|------|
| 首页 | Three.js 3D 粒子动画 |
| Toolbox | AI 工具的导航页面 |
| Key Checker | 检测 API Key 是否可用 |
| MBTI 测试 | 简单的性格测试工具 |
| 小游戏 | 网页端运行的格斗游戏 |

每个页面的实现都比较简单，但功能是完整的。我倾向于把工具做完、上线、验证效果，而不是花大量时间在设计打磨上。

## 移动端适配

网站的主要访问来源之一是微信内的浏览器。在微信中打开网页，如果页面有 JavaScript 报错，很多时候会直接白屏，用户什么也看不到。

这个问题在开发过程中遇到了几次。排查下来，大部分情况是某些 API 在微信浏览器中不支持，或者某些 polyfill 没有正确加载。解决办法也不复杂——在开发阶段就多在微信里测试，发现报错及时修复。

## 后续迭代

目前计划在网站上增加一个文章阅读模块。博客中的精选文章会同步过来，作为"推荐阅读"展示。这样主站有工具属性用来吸引流量，博客有内容属性用来沉淀读者，两者互相补充。

具体怎么做还没有完全想好，方向是先确定了。
