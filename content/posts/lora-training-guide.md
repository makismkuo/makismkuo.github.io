## 训练 LoRA 的起因

AI 生图发展到今天，论单张图片的质量已经相当不错了。但有一个问题始终存在——一致性。

如果你让 AI 画一个人物，今天画的是一张脸，明天改个 prompt 再画，出来的可能是完全不同的人。这在内容创作场景中是很麻烦的，比如你要做一个系列插图，主角需要长得一样。

LoRA 的解决办法是：用少量样本图片训练一个轻量模型，这个模型记录了人物的特征。出图时加载这个模型，AI 就知道"哦，原来这个人的脸长这样"。

## 样本准备：最关键的一步

这是整个过程中最重要的环节，但也是我一开始做得最差的一环。

第一次训练时，我找了大概 30 张照片，没有仔细筛选就扔了进去。照片里有侧脸的、有戴帽子的、有背光拍暗的、还有人占画面比例很小的。结果训练出来的 LoRA 效果很差——画出来的人物既像本人又不像本人，五官的位置是对的，但整体感觉就是不对。

第二次训练，我只用了 12 张照片，但每一张都是精心挑选的：正面拍摄、光线均匀、人物在画面中占比大、表情自然。出来的效果跟第一次天差地别。

这个对比让我意识到一个道理：LoRA 训练的效果上限，取决于你的样本质量。用 12 张高质量照片训练的效果，很可能好于 50 张随便收集的照片。

## API 调用的注意点

训练好 LoRA 之后，生成图片时调用 API 有一个容易踩的坑。

如果你的 LoRA 托管在 Replicate 这类平台上，调用的时候要用模型版本的 ID：

```bash
POST /v1/models/{owner}/{name}:{version}/predictions
```

有些教程里会说用 `lora_weights` 参数传入一个 URL。这个方法确实能用，但 CDN 生成的链接有时效性，几个小时到一天之后就会过期。一旦链接失效，你的图片生成请求就会返回 404。

第一次踩到这个坑的时候，我以为是服务器出了问题，排查了大半天才发现是链接过期了。后来把所有调用都改成了通过版本 ID 引用，问题才解决。

## 触发词

训练 LoRA 的时候需要指定一个触发词。这个触发词可以理解为一个"开关"——在 prompt 里带上这个词，AI 就会加载 LoRA 的权重。

我用的是 `TOK`，三个大写字母，不容易跟自然语言混淆。出图时在 prompt 里写 "a portrait of TOK, wearing glasses" 就会生成带有 LoRA 特征的人物。

## 成本

在 Replicate 上训练一次 LoRA 大概几美元，每次生成图片约 0.05 美元。如果用量不大，这个成本是可以接受的。对比 Midjourney 的订阅费，按需付费的方式在低频使用场景下其实更划算。

## 适合做什么

从实际使用的角度看，LoRA 适合以下场景：

- 固定角色设计：漫画、小说插图、品牌 IP
- 产品图生成：同一产品在不同场景下的展示
- 头像生成：用同一组特征生成多张不同风格的头像

如果你只是偶尔需要生成单张图片，LoRA 的学习成本和训练成本可能不太划算。但如果你需要大批量生成风格统一、角色一致的图片，投入这几个小时是值得的。
