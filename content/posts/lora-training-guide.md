## 需求

做角色设计的时候，最烦的一件事就是：AI 每次画出来的人长得不一样。

今天画的是一个样，明天改个 prompt 又换了一张脸。就像在跟一个脸盲的画家合作——他永远记不住主角长什么样。

LoRA 就是来解决这个问题的。它相当于给 AI 加了个"人物记忆模块"。

## 训练照片决定一切

这是最大的教训，也是最贵的经验。

第一次训练我随便找了 30 张照片扔进去——有侧脸的、有背光的、有糊成马赛克的。结果出来的 LoRA 效果嘛……怎么说呢，画出来的人既像又不像，像那个人的远方表亲，三次方的。

第二次我仔细挑了 **12 张**照片：全是清晰正脸、光线均匀、表情自然。出来的效果直接起飞。

**结论：15 张精品照片 > 50 张杂图。** 少就是多。

## API 调用的坑

训练完 LoRA，生成图片的时候要注意：

```bash
# ✅ 正确：用模型版本 ID
POST /v1/models/{owner}/{name}:{version}/predictions

# ❌ 错误：用 lora_weights URL
# CDN 链接几个小时后就过期了，到时候 404
```

第一次我用了 CDN 链接，第二天打开昨天的图片链接，全变成了 404。对着空气骂了十分钟。

## 成本

每次生成大概 $0.05，比 Midjourney 便宜不少。如果你只是做角色设计、产品图、或者头像，这个性价比很合适。

## 流程总结

1. 精选 10-15 张高质量照片
2. 指定触发词（我用 `TOK`）
3. 训练 LoRA
4. 出图时在 prompt 里带触发词

现在每次画同一个人物，终于不用重新生十次来"碰运气"了。
