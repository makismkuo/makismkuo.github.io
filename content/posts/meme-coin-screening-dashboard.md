做交易的第一步，是选币。

Solana 上每天冒出来的 meme coin 多到你根本看不完。靠推特刷 KOL 喊单？等你看到的时候要么被套要么已经被砸穿了。靠 DexScreener 手动翻？翻到眼花也不一定筛得出来能看的。

所以我给自己写了个选币面板。

## 选什么币

思路很简单：用 onchainos 这个链上数据 CLI，把刚上链的 meme coin 拉出来，按我定义的规则打分排序，一眼看出哪些值得看，哪些该跳过。

打分都是链上数据，不看团队、不看叙事、不看推特热度。六个维度：

- **买卖比 1h**（20分）—— 买的人多还是卖的人多，最直接的资金流向信号
- **Top10 集中度**（20分）—— 前10持仓占比太高就是庄控盘，越低越健康
- **LP 进度**（15分）—— bonding curve 走了多少，越接近完成说明 LP 越安全
- **市值阶段**（15分）—— 太小的盘子（<$5K）随时归零，太大的（>$500K）已经过了早期红利
- **老鼠仓风险**（15分）—— 狙击手+内幕+新钱包比例，偏高直接跳
- **Holder 分散度**（15分）—— 地址多且分得散，比几个大户压着好

满分 100。70 以上 BUY，40-69 WATCH，40 以下 AVOID。

逻辑不复杂，写在一个函数里，一百来行。但够用了。

## 架构

最简架构，三个文件：

```
backend/app.py      — Flask API，封装 onchainos CLI
frontend/index.html — 单页 SPA，纯原生 JS
nginx              — 静态文件 + API 反向代理
```

后端收到请求就跑 `onchainos memepump tokens` 拿列表，跑 `onchainos token report` 拿详情，JSON 返回。前端渲染暗色面板，按评分排序，点开看详情。

整个项目后端就 288 行 Python，前端去掉 K 线后 350 行 HTML+JS。部署在一台 HK 轻量云上，nginx 反代，Certbot 配了 HTTPS。

## 第一次：加功能

第一版做出来的时候，我是奔着"越多越好"去的。

加上了模拟交易——点一下买入，记录价格和数量，算盈亏。加上了统计面板——持仓汇总、胜率、总收益。加上了自动轮询——每 30 秒刷一次行情，实时检查止盈止损条件。

做的时候挺爽的。加一两个功能又花不了多少时间，为什么不加上？

结果是越来越卡。onchainos 接口本身不快，每次轮询要跑好几条 CLI 命令。在手机上打开（我经常在微信/浏览器里看），数据加载明显有延迟。而且大部分功能我根本不用——我不需要在选币面板里做模拟交易，也不需要在列表页看统计。

我骗自己说"万一以后用得上呢"，但实际上在做功能堆砌，不是在做工具。

## 第二次：做减法

用了一周后，我把它拆了。

删了数据库，删了模拟交易的所有 API 和前端面板，删了统计页，删了自动轮询。按钮从"刷新"改成"扫描"——你手动点一下，它扫一次，扫完就停。

K 线也去掉了。本来每打开一个详情页，还要多调一次 `market kline`。去掉之后，打开一个币只调一次 `token report`，列表页只调一次 `memepump tokens`。整站每次手动操作就两次链上调用。

做完之后舒服多了。页面干净了，加载快了，我自己用得也顺手了。

## 减法比加法难

做加法容易。多写几行代码、多放一个按钮，几分钟的事。

做减法难。你得承认之前花时间做的功能其实没用。你得删代码而不是加代码。你得说服自己"少即是多"不是一句口号，是真要动手砍东西。

这事没什么特别的道理，就是一次切身的体验：给自己做工具，最怕的不是功能不够，是功能太多以至于你不想打开它。

现在这个面板就是三样东西——代币列表、详情评分、手动扫描。够了。
