上个月帮朋友处理一批公众号文章的水印，一共二十多张图，每张右下角都盖着一个半透明的二维码。

我先是打开 PS、套索工具、选中、填充、保存……搞了七八张就崩溃了。又试了几个在线去水印站，传一张图等半分钟，一次只能一张，还没处理完先让你充值。

干脆自己搭一个。

前后折腾了三天，最后搞出来的东西，比我想象中好用不少。现在已经挂在 `aibestapp.top/clean/` 上自己用着，顺手把源码丢 GitHub 了。

**Fansen MarkGo** — 一个 AI 驱动的在线去水印工具。

---

## 它能干什么

一句话：上传图片 → 涂一下要去掉的地方 → 等几秒 → 下载干净的图。

听起来和那些在线去水印站差不多，但有几个不一样的点：

**🧹 涂哪去哪，不留痕迹**
底层跑的是 IOPaint 的 lama 模型，不是简单的像素填充或者模糊。涂掉的地方，AI 根据周围的纹理、色彩、光线重新生成。去完二维码之后那个位置看起来就是正常的背景，不会有"这里被涂过"的感觉。

**📱 微信里直接打开就能用**
这是最实用的一点。很多在线工具在微信浏览器里打不开，或者上传按钮没反应。Fansen MarkGo 的前端全是用 ES5 写的，兼容微信 X5 内核。你在微信里收到一张带水印的图，直接切到这个页面，传图、涂、下，全程不离开微信。

**📦 批量去水印，一次搞定**
经常处理公众号素材的人应该懂——水印通常在同一位置。你传第一张图涂好 mask，剩下的图共用同一个设置，一键批量处理完，打包 ZIP 下载。

**🖼️ 原图分辨率，不压缩**
有些网站为了省服务器资源，会先压缩你的图片再处理。Fansen MarkGo 传给 AI 的是原图文件，处理结果是完整分辨率。去完水印的图拿来打印或者发朋友圈都不会糊。

## 处理效果怎么样

拿几张实际的效果来看。

### 二维码去除

公众号文章底部最烦人的就是那个二维码，位置固定，半透明，底色白。直接涂掉，AI 补出来的效果基本看不出痕迹。

### 公众号 LOGO 水印

文章中间穿插的带 LOGO 水印图，涂掉之后背景如果是纯色或渐变，处理结果很干净。如果是复杂的照片背景，仔细看稍微能感觉一点处理痕迹，但日常使用完全够。

### 批量处理体验

二十张图，全部传上去，涂一次 mask，点批量处理。处理完一键打包下载 ZIP。隔壁同事还在 PS 里一张一张擦的时候，我已经搞定了。

## 技术是怎么搭的

部署结构不复杂：

```
前端 HTML → nginx → FastAPI 后端 → IOPaint 引擎 (lama 模型)
```

**前端**就是纯 HTML + CSS + JS，没有任何框架依赖。全部 ES5 语法，这是为了兼容微信浏览器。画 mask 用的 Canvas API，触摸和鼠标都支持。

**后端**是 Python FastAPI，收图片 → 把 mask 缩放到匹配原图尺寸 → 调 IOPaint API → 返回处理结果。附加功能包括水印自动检测（针对公众号右下角水印位置做了优化）、批量处理的实时进度推送、结果打包下载。

**AI 引擎**是 IOPaint，用的是 lama 模型。服务端 CPU 推理，一张 3000×4000 的图大概 10-30 秒。GPU 的话能进一两秒，后续可以升级。

**部署**在一台腾讯云香港轻量服务器上，nginx 反代，systemd 管理进程。

## 踩过的坑

### 第一个坑：手机微信里传图就崩溃

一开始前端用的是 `FileReader.readAsDataURL` 把图片转 base64 再发，想着这样后端处理方便。结果在手机上一张大图（iPhone 拍的三四千万像素）转成 base64 字符串直接爆内存，微信浏览器直接卡死。

后来改成直接发二进制 File 对象 + FormData，后端收二进制再转 base64，完全不卡了。

### 第二个坑：Canvas 尺寸对不上

前端画 mask 的 canvas 是 CSS 显示尺寸（手机上大约 390px 宽），但发给 IOPaint 的是原图全分辨率。image 和 mask 尺寸不匹配，IOPaint 直接报 400。

解决方案在后端：收到前端发来的全分辨率图片和 CSS 尺寸的 mask 后，用 Pillow 把 mask 缩放到匹配原图尺寸，再一起发给 IOPaint。

### 第三个坑：微信下载拦截

处理完的图片用 `window.open` 下载，微信直接拦截了。改成创建隐藏 `<a>` 标签触发点击，下载正常。

## 哪些场景值得用

**适合：**
- 公众号小编每天处理带水印的素材图
- 自媒体人批量处理多平台发布的图片
- 用手机处理图片但不想装 App 的人（微信里打开就用）
- 要高清原图、不想被压缩的人

**不太适合：**
- 需要批量处理几百张图的重度场景（CPU 推理速度有限）
- 水印覆盖了大面积复杂纹理的情况（AI 补出来的效果可能不够自然）
- 视频去水印（这个工具只处理静态图片）

---

## 说两句实话

这个工具离"完美"还很远。CPU 跑模型就是慢，全分辨率一张图十几秒是正常水平。如果你的场景是每天处理几十上百张，建议还是配张显卡，或者直接用桌面端工具。

但如果你跟我一样——偶尔处理一批图，图个方便，不想装软件、不想注册充值、不想被压缩画质——那这个东西可能正好对你有用。
