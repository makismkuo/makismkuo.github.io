## 你的 AI 令牌都去哪了

如果你在用 Claude Code、OpenAI Codex 或者任何 AI 编码助手做日常开发，你有没有想过一个问题：**我到底花了多少 token？**

我最近发现，自己每个月的 API 账单跟坐过山车一样。上个月 20 美元，这个月突然跳到 60 美元。仔细一想，可能是因为某个下午反复让 AI 改同一个函数的实现，每次改完不满意又重来，一个简单的重构吞掉了好几万 token。

问题的根源在于：**你看不到 token 的实时消耗。**

各大平台的控制台都提供用量统计，但都是滞后的——要么按天汇总，要么按小时出报表。等你看到账单的时候，钱已经花了。更关键的是，你无法把具体的 token 消耗和具体的操作关联起来：到底是哪个 prompt 花掉了我 2 万个 token？那个上下文窗口到底塞了多少东西？

今天我介绍一个项目，它刚好能解决这个问题。

## TokenTap 是什么

[TokenTap](https://github.com/jmuncor/tokentap) 是一个开源的 LLM 流量拦截工具，由开发者 jmuncor 用 Python 编写，基于 MIT 许可开源。它目前只有 **797 个 star**，属于那种躺在 GitHub 角落里的好东西。

它的工作方式很简单：**在你的本地启动一个 HTTP 代理，拦截所有发往 LLM API 的请求，实时统计 token 用量，然后在终端显示一个漂亮的仪表盘。**

```
┌─────────────────────────────────────────────────────────────┐
│  TOKENTAP - LLM Traffic Inspector                           │
├─────────────────────────────────────────────────────────────┤
│  Context Usage  ████████████░░░░░░░░░░░░░░░░  42%           │
│                 (84,231 / 200,000 tokens)                   │
├─────────────────────────────────────────────────────────────┤
│  Time     Provider    Model                      Tokens     │
│  14:23:01 Anthropic   claude-sonnet-4-20250514   12,847     │
│  14:23:45 Anthropic   claude-sonnet-4-20250514   8,234      │
│  14:24:12 Anthropic   claude-sonnet-4-20250514   15,102     │
├─────────────────────────────────────────────────────────────┤
│  Last Prompt: "Can you help me refactor this function..."   │
└─────────────────────────────────────────────────────────────┘
```

这个界面是实时的——你在另一个终端里跑 AI 工具，这边马上就能看到 token 在跳动。上下文窗口还有个颜色提示（绿色 < 50%、黄色 50-80%、红色 > 80%），一眼就能知道当前会话的用量情况。

## 核心功能

### 1. 实时仪表盘

用 Rich 库构建的 TUI 界面，色彩编码的上下文窗口燃料表，逐条显示的请求记录——时间、提供商、模型、token 数。你现在就能看到 AI 编码助手是不是在疯狂塞代码进上下文。

### 2. Prompt 存档

每次请求都会自动保存到本地目录，格式包括 Markdown（人类可读）和 JSON（原始 API 请求体）。想复盘某个 session 花了多少 token、发了什么 prompt，直接打开目录看就行。

### 3. 会话小结

退出的时候，终端会打出这次会话的总用量：`Session complete. Total: 84,231 tokens across 12 requests.` 数字很清楚。

### 4. 多提供商支持

目前支持 Anthropic（Claude Code）、OpenAI（Codex）、MiniMax。Gemini CLI 因为上游 bug 暂时不可用，但等修复了就能直接支持。

## 为什么值得关注

AI 编码助手确实提高了效率，但它们的 token 消耗是**隐形的**。你感受不到每次请求花了多少钱，因为你看到的只有终端里的代码输出。TokenTap 把这层窗户纸捅破了。

我用 Charlie 数了一下最近一周的 token 账单——原来"改几行代码"的请求平均要 8000 token，"写一个新函数"要 15000 token。而让 AI 帮我理解一个大型代码库的时候，一个请求能吞 30000+ token，因为连带着上下文一起发过去了。

有了 TokenTap，你能反过来优化自己的使用习惯：
- 发现上下文窗口快满了，主动清空重启
- 发现某个 prompt 意外地花了大量 token，调查原因
- 对不同的 AI 工具做 token 性价比对比

本质上，**TokenTap 给 AI 开发加了一层"成本可见性"**，没有这层可见性，你就是在闭着眼睛花钱。

## 安装和使用

安装只需要一行命令：

```bash
pip install tokentap
```

终端 1：启动仪表盘

```bash
tokentap start
```

终端 2：在你用的 AI 工具前面加上 tokentap 前缀

```bash
# Claude Code
tokentap claude

# OpenAI Codex
tokentap codex

# 或者指定提供商，运行任何命令
tokentap run --provider anthropic python my_app.py
```

就这么简单。没有证书配置，没有环境变量手写，不需要改你的 workflow。底层原理是一个 MITM 代理——tokenap 把 LLM API 的 base URL 指向本地代理，代理转发请求的同时截获并统计 token。你完全不用关心代理怎么配置。

## 和竞品的对比

市面上也有一些类似的 token 监控方案，但各有局限：

- **API 提供商的官方控制台**：延迟高（小时级报表），无法精确定位到某个 prompt
- **自建日志系统**：需要改造代码，在每次 API 调用的前后埋点统计
- **网络层抓包（Wireshark）**：太重了，搞到 token 数据还要自己解析 HTTP 请求体

TokenTap 的优势在于：**零侵入、零配置、实时呈现**。它工作在代理层，不需要改任何代码，安装即用。适合任何用 LLM API 做开发的场景，不光是 AI 编码助手，自定义的 LLM 集成也都能拦截。

当然它的局限也有——目前只支持 OpenAI 兼容的 API 格式和 Anthropic。如果你想监控其他 LLM 的流量，比如 Google 的 PaLM 或者 Cohere，得等作者或者社区贡献支持。

## 项目现状

TokenTap 是 2026 年 1 月才发布的项目，到现在不到 5 个月，正处于快速迭代期。代码质量不错，作者对 issue 的响应也挺积极。项目主要用 Python 和 Rich 库，代码结构和注释都很清晰，有兴趣的可以看看源码，不算复杂。

它的文档站点在 tokentap.ai，README 也有详细的命令说明和架构图。

## 结语

TokenTap 不是那种"改变世界"的项目，它是一个**解决具体痛点**的小工具。该痛点就是：AI 开发者花了钱但不知道钱花在哪。对于每天在终端里使用 AI 编码助手的开发者来说，这种可见性带来的价值远远超过装一个工具的成本。

用完 TokenTap 的第一天，我的最大感受是：原来之前每个月多付的钱，不是 AI 变贵了，是我自己用得太随意了。看到数字在仪表盘上跳动的时候，人会自然变得更谨慎、更理性。这可能比任何省 token 技巧都管用。
