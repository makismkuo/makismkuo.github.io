## 正则调试，一个老问题

写正则表达式的体验，大概可以分成三个阶段。

第一阶段是刚学的时候，对着文档一个一个查元字符，写出来的正则勉强能用。第二阶段是熟练了之后，能在脑子里跑简单的匹配，但遇到回溯、环视、原子分组这些概念还是发怵。第三阶段——绝大多数开发者的现状——打开 regex101.com，把正则贴进去，看高亮、看解释、看匹配结果。

regex101 很好用，但它有个问题：它是一个网页。这意味着你每次调试正则都要切到浏览器，意味着你在 SSH 连着的远程服务器上没法用，意味着你不能把结果 pipe 给其他命令。

rgx 解决的就是这个问题。

## rgx 是什么

rgx 是一个终端里的正则调试器，用 Rust 编写。它的核心定位是：**把 regex101 的交互体验搬到命令行里**。

项目地址：https://github.com/brevity1swos/rgx

目前 213 个 Star，发布不到半年，处于非常早期的阶段。但功能已经相当完整。

## 核心功能

rgx 的功能覆盖了正则调试的全流程，而且做得比预料中更深入。

**交互式 TUI**。运行 `rgx` 直接进入终端界面，上半部分写正则，下半部分写测试文本，实时显示匹配结果。AST 高亮、捕获组颜色区分、替换预览，基础操作都有。支持鼠标操作和 Vim 模式。

**三种正则引擎**。这是 rgx 最特别的设计。它同时支持 Rust 的内置 regex 引擎（最快、线性时间）、fancy-regex（支持环视和反向引用）和 PCRE2（支持递归和条件模式）。引擎会自动切换——你写了一个需要环视的模式，它自动从 Rust 引擎升级到 fancy-regex，完全无感。也可以用 `--engine` 参数手动指定。

**逐步调试器**。按 Ctrl+D 进入步进模式，逐步骤查看正则匹配过程，PCRE2 下还能看到回溯路径的可视化。这个功能对理解复杂正则的内部机制很有帮助，比如为什么某些模式在特定输入下会灾难性回溯。

**正则解释和代码生成**。按 E 键查看当前正则的逐段中文/英文解释，从 AST 自动生成。按 Ctrl+G 一键生成 Rust、Python、JavaScript、Go、Java、C#、PHP、Ruby 八种语言的代码片段，直接粘贴使用。按 Ctrl+U 还可以导出为 regex101.com 的共享链接。

**现场过滤模式**。`rgx filter` 模式可以把 stdin 或文件内容通过正则过滤，支持 JSON 字段提取。比如 `cat app.log | rgx filter 'error'` 交互式地过滤日志，调整正则时实时更新结果。

**基准测试**。Ctrl+B 一键对比三种引擎在该模式下的编译时间和匹配速度，对性能敏感的场景很有用。

## 为什么值得关注

首先，rgx 填补了一个真实空白。终端用户中正则的使用率很高，但之前没有一个像样的 TUI 调试器。你可以在 VS Code 的查找框里写正则，可以用 `grep -P` 在终端里跑正则，但**调试正则本身**的体验是缺失的。

其次，它是真正的跨平台——Linux、macOS、Windows 全支持，安装方式覆盖了 cargo、brew、AUR、预编译二进制。远程服务器上可以直接 `cargo install rgx-cli`。

第三，三引擎设计很有想法。不同场景适用不同的引擎——日常用最快的 Rust 引擎，需要环视和反向引用时自动升级，递归和条件模式才需要 PCRE2。这个渐进式设计让用户不需要操心引擎选择。

第四，代码生成和 regex101 导出是细节上的亮点。调试完正则直接生成代码，不用手动翻译到目标语言的正则写法。

## 使用示例

最简单的用法，进入交互模式：

```bash
rgx
```

启动时指定初始正则：

```bash
rgx '(\d{3})-(\d{3})-(\d{4})'
```

从 stdin 读取文本做匹配：

```bash
echo "联系我: 138-1234-5678" | rgx '(\d{3})-(\d{4})-\d{4}'
```

非交互模式，输出匹配结果到 stdout：

```bash
rgx -p 'error|warning' /var/log/app.log
```

过滤日志流：

```bash
tail -f app.log | rgx filter --json '.level' '(ERROR|FATAL)'
```

安装也很简单，Homebrew 用户一行搞定：

```bash
brew install brevity1swos/tap/rgx
```

或者用 Cargo：

```bash
cargo install rgx-cli
```

## 不足和展望

rgx 还很年轻。213 个 Star 说明它还在早期用户验证阶段。目前的一些限制：

- PCRE2 需要系统安装 `libpcre2-dev`，默认不启用
- 文档以英文为主，中文资料极少
- 编辑器的集成插件还在完善中

但对于终端重度用户来说，rgx 已经是一个可以日常使用的工具了。

## 结语

rgx 的出现让我想到一个观察：终端工具的进化远没有停止。在 AI 编码助手大行其道的今天，一个纯粹的正则调试器还能用 Rust 从零写出来，而且写得好用——这件事本身就说明，好的开发者工具依然有广阔的空间。

如果你也是一个终端用户，写正则的频率超过一周一次，rgx 值得试试。装上一分钟，调试正则的效率提升是实打实的。
