## 项目简介

[Harbor](https://github.com/av/harbor) 是一个用 Python 写的 CLI 工具，目标是让你**别再配置 AI 基础设施了，直接用就好**。一条 `harbor up` 命令就能拉起一套完整的本地 LLM 开发环境——Ollama + Open WebUI，预配好互联、预配好 RAG 搜索引擎，你打开浏览器就能聊天。

项目作者 `@av` 是 Docker 生态老手，Harbor 本质上是一个带 CLI 的 Docker Compose 编排层，但做对了极其关键的一件事：**所有服务默认互联、零配置**。目前在 GitHub 上已有 **3140+ star**，Apache-2.0 许可。

## 核心功能

Harbor 的能力远超"装个 Ollama + WebUI"的范畴。它管理的服务分三类：

- **后端推理引擎**：Ollama、llama.cpp、vLLM、TGI、SGLang、MLX（macOS Metal 原生）、oMLX、KTransformers、TabbyAPI、mistral.rs……你想到的推理框架基本都支持。
- **前端界面**：Open WebUI、LibreChat、ChatUI、Lobe Chat、AnythingLLM、Hollama、ComfyUI。
- **卫星服务**：SearXNG（搜索引擎）、Dify、n8n、Perplexica、Aider、Bolt.new、Home Assistant、Hermes Agent——70+ 个精选开源项目，一键启动。

每个服务都是独立 Docker 容器或主机进程，Harbor 自动配好网络、端口映射、模型缓存共享。启动新服务后，Open WebUI 里**直接就多了一个可用后端**，不需要进设置页面配 URL。

## 为什么值得关注

**痛点精准。** 用过本地 LLM 的开发者都知道这个场景：装好 Ollama → 装 Open WebUI → 配联网 → 想加 SearXNG 做 Web RAG → 手工写 docker-compose → 想试试 vLLM 的性能 → 又要改配置 → 想用 ComfyUI 出图 → 再来一遍。Harbor 把这一切压缩成一条命令：`harbor up ollama vllm comfyui`。

**多个独到设计：**

- **Harbor Boost**：一个优化的代理层，能提升小模型输出质量，支持自定义 workflow——deep research、deliverable audit、quick web lookup。
- **harbor launch**：自动把 Harbor 后端接入你本地的 AI 编码工具（Claude Code、Codex、OpenCode、Copilot、Hermes），一行命令配好 provider。
- **配置固化**：`harbor profile save` 保存当前后端参数（如 llama.cpp 的 `-ngl` 值），不同项目可快速切换。
- **harbor eject**：任何时候想独立运行，`harbor eject` 导出独立 docker-compose.yml，没有任何 vendor lock。

## 简单示例

```bash
# 一键启动默认环境（Ollama + Open WebUI）
harbor up

# 加 Web 搜索能力
harbor up searxng

# 加图片生成
harbor up comfyui

# 把 Ollama 后端接入 Claude Code
harbor launch --backend ollama --model qwen3.5:4b claude

# 保存一套针对大模型的配置
harbor llamacpp args -ngl 32 -c 8192
harbor profile save big-model
```

所有服务启动后，在 Open WebUI 里直接切换模型和工具，不需要离开浏览器。

## 总结

Harbor 不是一个"又一个 AI 工具"，而是**管理所有 AI 工具的工具**——一个本地 LLM 开发环境的入口。如果你正在本地跑模型、写 AI 应用、或者只是想无脑试各种推理框架，Harbor 能省去大量重复配置时间。开源、CLI 体验好、服务覆盖全面，是近期最值得关注的 AI 基础设施项目之一。
